# Guía rápida

## Uso recomendado para estudiantes

1. Leer el objetivo del bloque.
2. Revisar la formulación matemática.
3. Identificar los datos del caso.
4. Ejecutar el notebook de exploración.
5. Formular hipótesis antes de resolver el modelo.
6. Ejecutar el modelo entregado por el docente.
7. Comparar resultados con la interpretación esperada.
8. Completar la actividad de evaluación.

## Uso recomendado para docentes

1. Seleccionar un caso transversal, por ejemplo Garver.
2. Relacionarlo con distintos modelos: OPF, TNEP y GEP.
3. Entregar el `.mod` y `.run` en la plataforma institucional.
4. Mantener en GitHub los datos, guías, actividades y notebooks.
5. Usar la sección de IA para documentar prompts, validaciones y errores intencionales.

## Convención de carpetas

- `modelos/`: formulaciones matemáticas en Markdown.
- `datos/`: datos de prueba.
- `notebooks/`: notebooks de lectura, validación y visualización.
- `actividades/`: guías para estudiantes.
- `figuras/`: gráficos de apoyo.
