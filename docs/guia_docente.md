# Guía docente

[Menú principal](../README.md)

La secuencia de cada clase debe mantener una relación clara entre teoría, datos y modelación. El estudiante no debería comenzar escribiendo código: primero debe reconocer qué decisión se toma, qué información está disponible y qué restricciones hacen que el problema sea eléctrico y no solo numérico.

## Uso recomendado

1. Iniciar con el README del módulo y discutir el problema técnico.
2. Derivar la formulación en pizarra: conjuntos, parámetros, variables, función objetivo y restricciones.
3. Revisar los datos disponibles y sus unidades.
4. Pedir que el estudiante construya el `.dat` antes de escribir el `.mod`.
5. Implementar el modelo en AMPL o el flujo de datos en Python, según corresponda.
6. Ejecutar una variación de sensibilidad: demanda, reserva, costo, límite de línea, disponibilidad o VOLL.
7. Cerrar con interpretación técnica, no solo con el valor de la función objetivo.

## Productos de clase

Cada actividad debe producir, como mínimo, una formulación, un archivo de datos, resultados tabulados, una figura útil y una explicación de consistencia. En los módulos de optimización, operación, OPF, TNEP y GEP, el estudiante debe construir sus propios archivos `.mod`, `.dat` y `.run` a partir de la formulación del módulo.
